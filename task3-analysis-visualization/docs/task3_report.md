# 任务三：数据分析可视化智能体 — 技术报告

***

## 1. 概述

任务三目标是基于任务一（ETL数据处理）和任务二（知识图谱问答）的成果，构建一个**数据分析与可视化智能体**。用户提出自然语言分析需求（如"糖尿病有什么症状分布？"），智能体自动完成统计分析→图表可视化→洞察报告的全流程。

### 核心理念

> **NL→结构化查询（非真SQL）**：赛题要求"NL2SQL准确率≥85%"，但我们没有SQL数据库。我们的"SQL"是**结构化图谱查询指令**——LLM将用户意图翻译为`kg_analyze`的参数，工具返回统计数据，LLM再驱动可视化。

### 链路闭环

```
任务二KG三元组 → kg_analyze(统计分析) → kg_visualize(图表) → kg_report(洞察报告)
```

## 3. 三个MCP工具

### 3.1 kg\_analyze — 图谱统计分析

**职责**：对KG三元组做统计分析（计数/分组/排序/占比）

| 分析类型 | 输入             | 输出示例                      |
| :--- | :------------- | :------------------------ |
| 关系分布 | `entity="糖尿病"` | `[{item:"导致",count:66}]`  |
| 症状分布 | `entity="糖尿病"` | `[{item:"多饮多尿",count:8}]` |
| 药物统计 | `entity="高血压"` | `[{item:"硝苯地平",count:1}]` |
| 关联分析 | `entity="糖尿病"` | 关联所有实体计数                  |

**四层架构**：

```
基础设施层(筛选过滤) → 统计处理器层(4种分析) → 引擎层(映射表分发) → MCP入口
```

**关键设计**：参数覆盖逻辑——"症状分布"自动使用 relation="导致"，"药物统计"自动使用 relation="治疗"，防止LLM传错参数。

### 3.2 kg\_visualize — 图表可视化

**职责**：将统计结果渲染为ECharts HTML图表

| 图表类型        | 适用场景      | 输出                 |
| :---------- | :-------- | :----------------- |
| `bar` 柱状图   | 症状/药物分布统计 | HTML文件（交互式ECharts） |
| `pie` 饼图    | 关系类型占比    | HTML文件             |
| `graph` 关系图 | 知识图谱网络展示  | HTML文件（力导向布局）      |
| `line` 折线图  | 趋势分析      | HTML文件             |

**安全性设计**：

- 分层转义：HTML上下文用 `_escape_html()`，JS上下文用 `_escape_js()`
- 白名单验证：width/height正则校验，theme枚举校验
- CDN降级：`onerror` 捕获ECharts加载失败

### 3.3 kg\_report — 分析报告生成

**职责**：整合统计+图表，输出结构化Markdown报告（含洞察+置信度）

**三种报告模式**：

| 模式              | 内容             | 场景   |
| :-------------- | :------------- | :--- |
| `full` 完整版      | 概述+统计+图表+洞察+尾部 | 综合查询 |
| `summary` 摘要版   | 核心发现+关键数据      | 快速回答 |
| `technical` 技术版 | 原始数据+统计详情      | 技术展示 |

**洞察规则引擎（8条规则）**：

|  #  | 规则      | 触发条件              |   severity   |
| :-: | :------ | :---------------- | :----------: |
|  1  | 高集中度    | Top-1占比>35%       | 🔴 highlight |
|  2  | 均衡分布    | Top1-Top2差距<5%    |    🔵 info   |
|  3  | Top-3集中 | Top-3合计>60%       |    🔵 info   |
|  4  | 长尾显著    | Top-3合计<40%       |    🔵 info   |
|  5  | 禁忌警告    | 存在禁忌关系            |  🟡 warning  |
|  6  | 数据稀疏    | total\_triples<5  |  🟡 warning  |
|  7  | 数据丰富    | total\_triples>30 |    🔵 info   |
|  8  | 跨分析因果链  | 症状+药物同存           | 🔴 highlight |

**创新点**：

- **置信度评分引擎**：每条洞察附带 `confidence: 0-1` 分，结合样本量和效应量动态调整
- **冲突消解**：规则1和2互斥（高集中度优先），规则3和4互斥（集中优先）
- **跨分析因果链**：如症状分布+药物统计同时存在，自动检测"症状→治疗"模式

## 4. 智能体设计与Prompt

### 4条处理路径

| 路径         | 场景          | 流程                           |
| :--------- | :---------- | :--------------------------- |
| **A** 事实查询 | "糖尿病有什么症状？" | analyze→visualize→report     |
| **B** 关系概览 | "全面分析一下糖尿病" | analyze×3→visualize×2→report |
| **C** 用药分析 | "高血压用什么药？"  | analyze(药物)→visualize→report |
| **D** 对比分析 | "A和B有什么区别？" | analyze×2→visualize×2→report |

### Prompt三段式

```
duty_prompt → 角色定义 + 3工具说明 + 4条路径 + 调用示例
constraint_prompt → 8条规则（格式/数据传递/错误处理/禁止编造）
few_shots_prompt → 3个完整示例（症状查询/药物查询/综合分析）
```

## 6. 创新点

1. **三层架构的图谱分析工具链** — kg\_analyze(统计) → kg\_visualize(图表) → kg\_report(洞察)，分工明确，各司其职。评委可在浏览器中直接打开ECharts交互图表，无需额外工具
2. **规则驱动的洞察引擎** — 不依赖LLM，8条规则从数据中自动提炼关键发现（如"某症状是最突出的表现"），每条洞察附带置信度评分和可溯源引用，增强可信度
3. **跨分析因果链检测** — 能识别症状分布和药物统计之间的关联关系，自动发现"症状→治疗"的因果模式，将孤立的统计数据关联为有意义的医学洞察
4. **四路LLM编排策略** — 智能体根据用户意图自动选择路径（症状查询/关系概览/用药分析/对比分析），而非固定流程——这是真正的"智能体"而非"脚本执行器"
5. **零依赖安全可视化** — HTML+ECharts CDN零外部Python依赖，分层转义策略（\_escape\_html + \_escape\_js + 白名单）彻底杜绝XSS和CSS注入风险
