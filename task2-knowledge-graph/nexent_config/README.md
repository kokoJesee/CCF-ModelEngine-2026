# 医疗知识图谱问答智能体 v2.5 - Nexent部署指南

## 📋 概述

本指南介绍如何在Nexent平台上部署医疗知识图谱问答智能体v2.5。采用**双代理验证架构**，支持超级节点过滤和上下文约束推理。

### 架构特点

- **双代理验证**：推理代理 + 验证代理（协作智能体），确保回答准确性
- **超级节点过滤**：自动过滤高频症状节点（发热、乏力、头痛、呕吐）
- **上下文约束**：基于患者个体差异进行个性化推理
- **经验记忆**：学习历史案例，持续优化回答质量

### 双代理架构

```
                用户问题
                    │
                    ▼
    ┌───────────────────────────────┐
    │   推理代理 (kg_qa_reasoning)   │ ← deepseek-v4-flash
    │   - 问题解析                   │
    │   - 知识图谱检索               │
    │   - 推理与整合                 │
    │   - 结构化输出                 │
    └───────────┬───────────────────┘
                │ 推理结果
                ▼
    ┌───────────────────────────────┐
    │   验证代理 (kg_qa_verification)│ ← deepseek-v4-flash
    │   - 事实准确性验证             │
    │   - 逻辑合理性验证             │
    │   - 安全性评估                 │
    │   - 质量评分 (阈值≥0.7)       │
    └───────────┬───────────────────┘
                │ 修订建议/通过
                ▼
          最终回答
```

## 🚀 快速部署（三步搞定！）

### 第一步：配置MCP服务器

在Nexent后台添加MCP服务器，指向你的DataMate服务：

1. 进入 **智能体开发** → 选择智能体 → **工具配置**
2. 点击 **MCP配置** → **Add MCP Server**
3. 填写：
   - **Server name**: `datamate_mcp`
   - **Server URL**: `http://host.docker.internal:8089`
4. 点击 **+ Add** 完成添加
5. 测试连接状态，确认显示"可用"

### 第二步：一键导入智能体

1. 进入 **智能体开发** → 点击 **导入 Agent**
2. 选择本目录下的 `kg_qa_agent_nexent_v2.5_import.json`
3. 系统自动验证配置格式，显示导入的智能体信息
4. 如提示重名，选择 **重新生成并导入**
5. 完成导入后，在智能体列表中看到两个智能体：
   - `kg_qa_reasoning_agent`（主智能体 - 推理代理）
   - `kg_qa_verification_agent`（协作智能体 - 验证代理）

### 第三步：检查配置

导入后建议检查以下配置项：

| 配置项         | 预期值                                                                                   |
| :---------- | :------------------------------------------------------------------------------------ |
| **大语言模型**   | deepseek-v4-flash（需先在模型管理中配置好API Key）                                                     |
| **最大步骤数**   | 推理代理: 15 / 验证代理: 10                                                                   |
| **协作Agent** | 推理代理已关联验证代理                                                                           |
| **MCP工具**   | 4个工具（knowledge\_graph\_search, entity\_recognition, relation\_extraction, datamate工具） |
| **智能体描述**   | 包含完整的业务逻辑描述                                                                           |

> ⚠️ 如果导入后模型显示为空，请先在 **设置 → 模型管理** 中配置 `deepseek-v4-flash` 的API Key

## 🔧 配置详解

### 智能体变量命名规范

| 智能体         | 变量名                        | 说明              |
| :---------- | :------------------------- | :-------------- |
| **推理代理**（主） | `kg_qa_reasoning_agent`    | 处理用户问题，进行知识图谱推理 |
| **验证代理**（子） | `kg_qa_verification_agent` | 验证推理结果，提供质量评估   |

### 模型配置

| 参数       | 值                               | 说明              |
| :------- | :------------------------------ | :-------------- |
| 主模型      | `deepseek-v4-flash`                 | 推理和验证使用同一模型     |
| API Base | `https://api.siliconflow.cn/v1` | 需在Nexent模型管理中配置 |
| 温度       | 0.3                             | 低温度保证一致性        |
| 最大Token  | 4096                            | 足够处理复杂医疗推理      |

### MCP工具说明

| 工具名                          | 对应算子                 | 功能          | 使用方   |
| :--------------------------- | :------------------- | :---------- | :---- |
| `knowledge_graph_search`     | `kg_search`          | 查询知识图谱实体和关系 | 推理+验证 |
| `entity_recognition`         | `entity_recognizer`  | 识别医疗文本中的实体  | 推理+验证 |
| `relation_extraction`        | `relation_extractor` | 抽取实体间关系     | 推理+验证 |
| `datamate_verify_connection` | -                    | 验证MCP连接状态   | 推理代理  |
| `datamate_list_operators`    | -                    | 列出可用算子      | 推理代理  |

### 提示词结构

Nexent导入JSON使用三段式提示词结构：

| JSON字段              | 内容来源               | 说明            |
| :------------------ | :----------------- | :------------ |
| `duty_prompt`       | 角色定位 + 工作流程        | 定义智能体的角色和核心职责 |
| `constraint_prompt` | 边界条件 + 质量标准 + 使用说明 | 约束智能体的行为边界    |
| `few_shots_prompt`  | 输出格式 + 示例          | 提供具体的输出示例参考   |

导入完成后，在Nexent中进入智能体的**调试**模式，进行测试：

### 基础测试

```text
用户：发热可能是什么原因？
```

预期行为：

1. 推理代理识别实体"发热"
2. 触发超级节点过滤机制
3. 检索知识图谱中发热相关的疾病
4. 输出结构化推理结果

### 上下文约束测试

```text
用户：我父亲65岁，有2型糖尿病和轻度肾功能不全（eGFR 45），
      最近血糖控制不好，有什么治疗建议？
```

预期行为：

1. 推理代理提取实体和约束条件
2. 检查药物禁忌（eGFR < 30时某些药物禁用）
3. 验证代理审核推理结果的完整性和安全性
4. 输出经过验证的最终答案

## ❗ 常见问题

### Q1: 导入JSON时提示"格式错误"？

A: 确认导入的是 `kg_qa_agent_nexent_v2.5_import.json` 文件，不是旧版的 `kg_qa_agent_nexent_v2.5.json`。新文件完全符合Nexent的 `ExportAndImportDataFormat` 格式。

### Q2: MCP工具不可用？

A: 必须先添加MCP服务器（Server name: `datamate_mcp`），再导入智能体。如果MCP服务器未在"工具配置"中注册，AI不会自动识别这些工具。

### Q3: 显示"模型未配置"？

A: 在 **设置 → 模型管理** 中添加 `deepseek-v4-flash` 模型，配置对应的API Key。导入JSON只记录了模型名，不会包含API Key。

### Q4: 两个智能体的关系？

A: `kg_qa_reasoning_agent` 是主智能体，`kg_qa_verification_agent` 是其协作智能体。在Nexent中运行主智能体时，它会自动调用验证代理进行审核。

### Q5: 算子还没开发完怎么办？

A: 可以先导入测试智能体的基础对话能力（不带知识图谱检索）。Week 5完成算子开发并上传DataMate后，MCP工具会自动可用。

***

## 📝 注意事项

1. **MCP服务器必须先配置**，否则导入后工具不可用
2. **API Key不会包含在导入JSON中**，需在Nexent模型管理中单独配置
3. 导入重复名称时，建议选择"重新生成并导入"以避免冲突
4. 知识库工具（`knowledge_base_search` 等）的配置不会从导入JSON继承，需在Nexent中重新配置知识库权限
5. 导入后建议在调试模式中测试完整流程
